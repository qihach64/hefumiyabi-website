### 1.2 设计亮点 ✅

| 功能 | 描述 | 我们可借鉴 |
|------|------|-----------|
| **货币转换** | 实时汇率转换 (JPY/TWD/HKD/CNY) | ⭐⭐⭐⭐⭐ 必须有 |
| **预订流程可视化** | 4步骤图示 + "2分钟完成" | ⭐⭐⭐⭐ 降低决策成本 |
| **发型选项展示** | 基础/升级发型分层，带实拍图 | ⭐⭐⭐⭐ 增值服务展示 |
| **店铺差异化** | 按客群推荐 (新手/摄影/古街) | ⭐⭐⭐⭐ 帮助用户选择 |
| **信任指标** | "每年超过23万人使用" | ⭐⭐⭐⭐ 社会证明 |
| **FAQ 嵌入** | 常见问题直接展示在页面 | ⭐⭐⭐ 减少咨询 |
| **取消政策醒目** | "前一天可免费取消" | ⭐⭐⭐ 降低预订顾虑 |
| **多 CTA 按钮** | 红色"预订此方案"出现多次 | ⭐⭐⭐ 提升转化 |

### 1.3 设计缺点 ❌

| 问题 | 描述 | 我们的机会 |
|------|------|-----------|
| **无 AI 试穿** | 只有静态穿着效果图 | **我们的核心差异化** |
| **无互动地图** | 无法了解和服配件构成 | **我们的 InteractiveKimonoMap** |
| **图片质量参差** | 部分图片像素低/风格不统一 | 高质量视觉呈现 |
| **无用户评价** | 缺乏真实用户反馈 | 买家秀 + 评价系统 |
| **排版较拥挤** | 信息密度高，留白少 | Airbnb 风格留白 |
| **无锚点导航** | 长页面滚动困难 | AnchorNav 导航 |
| **设计感偏传统** | 功能导向，缺乏品牌调性 | Japanese Modernism |

### 1.4 核心学习点

**必须借鉴**:
1. 💱 **货币转换功能** - 游客刚需
2. 📋 **预订流程可视化** - "只需X步，约X分钟完成"
3. 💇 **增值服务分层展示** - 基础/升级选项清晰
4. 🏪 **店铺差异化推荐** - 按场景/客群匹配
5. ❓ **FAQ 嵌入页面** - 减少跳转

**我们的差异化优势**:
1. 🤖 **AI 试穿** - 竞品完全没有
2. 🗺️ **交互式和服配件图** - 教育性内容
3. 🎨 **设计品质** - Japanese Modernism vs 传统电商
4. ⭐ **用户评价系统** - 社会证明

## 2. 内部原型分析：kimono-frontend

> **来源**: 内部设计原型项目
> **设计理念**: "一主题一世界" 的沉浸式杂志化体验

### 2.1 页面结构对比

| 模块 | kimono-frontend | hefumiyabi-website | 差异 |
|------|-----------------|-------------------|------|
| **Hero/头图** | 60vh 全屏沉浸式 Hero + 视频背景支持 | Airbnb 风格 2+4 网格画廊 | kimono 更沉浸，hefumiyabi 更实用 |
| **导航** | 锚点导航栏 (AnchorNav) | 无 | kimono 有页内导航 |
| **布局** | 双栏 (8:4) | 双栏 (2:1) | 类似 |
| **预订卡片** | Sticky 侧边栏 + 抽屉 Drawer | Sticky 侧边栏 | kimono 有 Drawer 交互 |
| **图片画廊** | 虚拟滚动 GalleryGrid | ImageGalleryModal | hefumiyabi 有 Lightbox |
| **旅程预览** | JourneyTimeline 时间轴 | 无 | kimono 独有 |
| **买家秀** | InspirationWall 虚拟滚动 | 静态评价列表 | kimono 更丰富 |
| **AI 试穿** | 无 | TryOnModal + ImageComparison | hefumiyabi 独有 |
| **和服配件图** | 无 | InteractiveKimonoMap | hefumiyabi 独有 |

### 2.2 核心设计理念差异

**kimono-frontend**:
- "一主题一世界" 的沉浸式杂志化体验
- Desktop Only (1280px+)，拒绝移动端妥协
- 注重视觉冲击力和动效
- Command Palette (Cmd+K) 全局联动

**hefumiyabi-website**:
- Airbnb 风格的实用主义设计
- 响应式设计，兼顾移动端
- 注重预订转化和功能完整性
- AI 试穿作为差异化卖点

### 2.3 kimono-frontend 设计亮点

#### 2.3.1 沉浸式 Hero 区域

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│     60vh 全屏 Hero                                      │
│     视频/图片背景 + 渐变遮罩                             │
│                                                         │
│     ┌──────────────────────┐                            │
│     │ 浅草情侣和服套餐      │  ← 大标题 (font-serif)     │
│     │ 专为情侣设计的浪漫... │  ← 描述                    │
│     │ 📍 浅草寺店  ⏱ 4小时 │  ← 元信息                   │
│     └──────────────────────┘                            │
└─────────────────────────────────────────────────────────┘
```

**优点**: 视觉冲击力强，建立品牌调性
**缺点**: 占用大量首屏空间，可能影响转化

#### 2.3.2 锚点导航栏 (AnchorNav)

```
┌─────────────────────────────────────────────────────────┐
│  视觉  │  概览  │  旅程  │  买家秀  │  店铺             │
└─────────────────────────────────────────────────────────┘
         ↑ Sticky 固定在 Navbar 下方
```

**功能**:
- 点击滚动到对应区块
- Scroll Spy 高亮当前区块
- 提升长页面导航体验

#### 2.3.3 旅程预览 (Journey Timeline)

```
09:00 ──●── 到店 (Arrival)
         │   📍 距离车站步行5分钟
         │
09:15 ──●── 选衣 (Styling)
         │   👘 500+ 款式任选
         │
09:45 ──●── 变身 (Transformation)
         │   💇 专业发型师服务
         │
10:30 ──●── 漫步 (Stroll)
         │   🏯 推荐：清水寺、花见小路
         │
17:30 ──●── 归还 (Return)
             ⚠️ 请于18:00前归还
```

**价值**:
- 将"体验"可视化，而非只列功能
- 帮助用户预期时间安排
- 提升预订信心

#### 2.3.4 预订抽屉 (Booking Drawer)

```
┌──────────────────────┬─────────────────┐
│                      │    预订抽屉     │
│    主页面内容         │                │
│    (可滚动浏览)       │  Step 1: 日期  │
│                      │  Step 2: 人数  │
│    ← 半透明遮罩       │  Step 3: 升级  │
│                      │                │
│                      │  [立即预订]     │
└──────────────────────┴─────────────────┘
```

**特点**:
- 右侧滑出，不完全遮挡内容
- Auto-Save: 表单状态存入 localStorage
- 防误触: Dirty State 时关闭需确认

#### 2.3.5 主题差异化布局

| 主题 | Gallery 风格 | 字体 | 装饰 | 动效 |
|------|-------------|------|------|------|
| Formal | Standard (规整) | 宋体 | 金箔纹理 | Slow Fade |
| Trendy | Mosaic (错落) | 粗黑体 | 高饱和色块 | Spring |
| Seasonal | Cinematic (视频) | 细体 | 樱花粒子 | Parallax |
| Couple | Standard (柔和) | 圆体 | 双色渐变 | Blur |

## 4. 可借鉴的设计模块

### 4.1 高优先级 (High Impact, Low Effort)

#### A. 锚点导航栏 (AnchorNav)

**当前问题**: 页面较长，用户难以快速定位

**建议实现**:
```tsx
<AnchorNav sections={[
  { id: 'gallery', label: '图片' },
  { id: 'overview', label: '概览' },
  { id: 'kimono-map', label: '和服配件' },
  { id: 'reviews', label: '评价' },
  { id: 'booking', label: '预订' },
]} />
```

**实现难度**: ⭐⭐ (2/5)
**用户价值**: ⭐⭐⭐⭐ (4/5)

#### B. 旅程预览 (Journey Timeline)

**当前问题**: 用户不清楚实际体验流程

**建议实现**:
```tsx
interface TimelineNode {
  time: string;
  title: string;
  description: string;
  icon: 'arrival' | 'styling' | 'photo' | 'return';
  thumbnail?: string;
}

// 数据可存储在 RentalPlan 或单独的 PlanTimeline 表
```

**实现难度**: ⭐⭐⭐ (3/5)
**用户价值**: ⭐⭐⭐⭐⭐ (5/5)

### 4.2 中优先级 (Medium Impact)

#### C. 买家秀灵感墙 (Inspiration Wall)

**当前问题**: 评价区域缺乏视觉吸引力

**建议升级**:
- 添加买家秀图片 (用户上传的穿着照)
- 支持标签筛选: [粉色系] [情侣] [短发适配]
- 虚拟滚动优化性能

**数据模型扩展**:
```prisma
model Review {
  // ... existing fields
  photos      String[]     // 买家秀图片
  tags        String[]     // 风格标签
}
```

**实现难度**: ⭐⭐⭐⭐ (4/5)
**用户价值**: ⭐⭐⭐⭐ (4/5)

#### D. 预订抽屉 (Booking Drawer)

**当前问题**: BookingCard 功能完整但交互较传统

**建议升级**:
- 点击"立即预订"打开右侧抽屉
- 分步填写: 日期 → 人数 → 升级选项
- Auto-Save 到 localStorage
- 防误触确认弹窗

**实现难度**: ⭐⭐⭐ (3/5)
**用户价值**: ⭐⭐⭐ (3/5)

### 4.3 低优先级 (Nice to Have)

#### E. 主题差异化布局

**考量**: hefumiyabi 已有主题系统 (themeColor)，但布局差异化成本较高

**建议**:
- 保持当前统一布局
- 仅通过颜色和装饰元素区分主题
- 未来如有强烈需求再考虑

#### F. 沉浸式 Hero

**考量**:
- 当前 Airbnb 风格图片画廊实用性强
- 沉浸式 Hero 可能影响移动端体验
- 视频资源增加运营成本

**建议**: 暂不实现

## 5. 讨论点与决策建议

### 5.1 开放讨论

**Q1: 是否需要锚点导航？**
- 当前页面长度约 4-5 屏
- 移动端是否适用？
- 是否会与 Header 冲突？

**Q2: 旅程预览的数据来源？**
- 所有套餐共用一套流程？
- 还是每个套餐自定义？
- 如何在后台管理？

**Q3: 买家秀的运营成本？**
- 如何获取用户的穿着照片？
- 是否需要审核机制？
- 标签系统如何设计？

**Q4: AI 试穿与买家秀的关系？**
- 试穿结果是否可转为"买家秀"？
- 用户是否愿意分享？

### 5.2 我的建议

基于当前产品阶段和用户需求，建议采用**渐进式升级**策略：

```
Phase 1 (快速见效):
├── 添加 AnchorNav 锚点导航
└── 优化评价区域 UI

Phase 2 (提升体验):
├── 实现 JourneyTimeline 旅程预览
└── 升级为 BookingDrawer 抽屉交互

Phase 3 (丰富内容):
├── 买家秀灵感墙
└── AI 试穿结果分享功能
```

## 7. 内容模块整合方案

> **v1.2 新增** - 基于对竞品、内部原型和当前实现的分析，提出内容模块整合方案，减少冗余、提升信息密度。

### 7.1 问题分析：当前内容模块的冗余

当前设计中存在多个功能相似但分散的模块：

| 冗余类型 | 涉及模块 | 问题 |
|---------|---------|------|
| **图片展示重复** | 画廊 + 买家秀 + AI试穿 + 发型选项 | 4处图片展示，用户需跳转浏览 |
| **服务内容分散** | 套餐包含 + 和服配件图 + 升级选项 | 同一主题信息碎片化 |
| **社交证明割裂** | 用户评价 + 买家秀 + 评分 | 评价自带图片，买家秀可合并 |
| **流程信息冗余** | 预订流程 + 旅程预览 + FAQ + 须知 | 时间线可嵌入FAQ |

### 7.2 整合方案：4大核心模块
